## 狀態流程圖範例 (以一個完整療程包為例)

1. **老客戶的固定週期療程單的新單 (如 1/5)**：
    - 在服務開始前：狀態為 **「新訂單」**。
    - 服務進行中：狀態為 **「進行中」**。
    - 服務完成後（但整個5次的療程還沒結束）：狀態應變更為 **「待回診 / 週期中」**。並在訂單備註或相關欄位標註 `1/5`。
2. **週期中的回診單 (如 3/5)**：
    - 客戶再次到店時，將上一狀態 **「待回診 / 週期中」** 的訂單取出，重新激活為 **「進行中」**。完成後再變回 **「待回診 / 週期中」**，並標註 `3/5`。
3. **兩種「已完成」的區分**：
    - 為了避免混淆，建議不要將兩種都叫做「已完成」。
    - 將**服務操作完畢**的稱為 **「已完成 (服務面)」**。
    - 將**財務結算、毛利計算完畢**的稱為 **「已結算」**。
    - 將**庫存扣除完畢**的稱為 **「已扣帳」**。
    - 通常流程是：`服務完成` -> `財務審核/計算毛利` -> `自動/手動扣除庫存`。最後一個步驟完成後，該訂單在業務上才算真正「結束」。

### 狀態流程圖範例 (以一個完整療程包為例)

1. `預約待確認` -> (客戶確認/付訂) -> `新訂單` (1/5)
2. (客戶到店) -> `進行中`
3. (服務結束，5次未完成) -> `待回診` (標註 1/5)
4. (客戶回診做第二次，2/5) -> `進行中`
5. (服務結束，5次未完成) -> `待回診` (標註 2/5)
6. ... (循環步驟4-5)
7. (客戶做第五次，5/5) -> `進行中`
8. (第五次服務結束) -> `已完成 (服務面)`
9. (財務人員確認金額) -> `已結算`
10. (系統扣除本次使用的細胞膜、針劑等庫存) -> `已扣帳`



## 從「已完成 (服務面)」到「已結算」再到「已扣帳」的狀態變更

### 建議方案：採用「操作按鈕 + 確認彈窗」機制

**核心原則：**

- **禁止直接下拉修改：** 不允許用戶從下拉選單直接將「已完成」改為「已扣帳」。
- **按鈕驅動流程：** 每個狀態變更都必須通過獨立的按鈕觸發，並在後台執行對應的邏輯（計算毛利、扣庫存）。
- **權限分離：** 只有財務人員能看到「結算」按鈕，只有庫管人員能看到「扣帳」按鈕。

### 1. 介面設計 (UI/UX)

在訂單詳情頁或訂單列表的操作欄中，根據當前狀態動態顯示按鈕：

- **當訂單狀態為「已完成 (服務面)」時：**
    - 顯示一個藍色的 **【結算訂單】** 按鈕。
    - *不顯示*「扣帳」按鈕（因為尚未結算，還不知道毛利，不應扣帳）。
- **當訂單狀態為「已結算」時：**
    - 顯示一個綠色的 **【確認出貨/扣帳】** 按鈕。
    - 此時，財務相關欄位（毛利、毛利率）應鎖定不可編輯。
- **當訂單狀態為「已扣帳」時：**
    - 所有操作按鈕變灰或隱藏，僅顯示「查看詳情」。

### 2. 後台邏輯與庫存/儀表板更新機制

為了確保庫存數據被更新，我們需要在點擊按鈕時觸發一系列後台動作。

**步驟 A：從「已完成」 -> 「已結算」**

1. **操作：** 財務人員點擊「結算訂單」。
2. **彈窗確認：** 彈出一個視窗，顯示該訂單的「收入」和「費用小計」，並自動計算出「毛利」。
    
    ```
    [結算確認]
    訂單編號：D2D-005
    收入：30,000
    費用小計：6,000
    -------------
    計算毛利：24,000
    毛利率：80%
    是否確認結算？
    [取消] [確認結算]
    ```
    
3. **後台動作：**
    - 將訂單狀態更新為「已結算」。
    - 將計算出的毛利寫入資料庫。
    - **更新財務儀表板：** 觸發一個事件，讓總收入、總毛利等圖表數據即時刷新（或標記為待刷新）。

**步驟 B：從「已結算」 -> 「已扣帳」**

1. **操作：** 庫管人員點擊「確認出貨/扣帳」。
2. **彈窗確認（關鍵步驟）：** 此時必須彈出一個**庫存變更預覽頁面**。
    - 系統自動抓取該訂單使用的所有藥材項目（例如：打針 -> NK）。
    - 顯示即將減少的庫存數量。
    
    ```
    [庫存扣帳確認]
    以下項目即將從庫存中扣除：
    
    | 藥材名稱 | 使用數量 | 當前庫存 | 扣帳後庫存 |
    | -------- | -------- | -------- | ---------- |
    | NK針劑   | 2        | 50       | 48         |
    | 細胞膜60 | 6        | 30       | 24         |
    
    警告：此操作不可逆，請確認實物已出庫。
    [取消] [確認扣帳]
    ```
    
3. **後台動作：**
    - 執行資料庫事務（Transaction）：先鎖定庫存表，扣除數量，再更新訂單狀態。
    - 若庫存不足（例如 NK 只剩 1 支，但訂單需要 2 支），系統應跳出錯誤提示：「庫存不足，無法扣帳」，並阻止狀態變更。
    - **更新庫存儀表板：** 即時更新庫存儀表板上的「庫存量」、「低庫存預警」等指標。

### 3. 如何確保庫存已更新？—— 雙重驗證機制

為了讓管理者放心，建議設置以下幾種視覺化反饋：

1. **訂單時間軸 (Activity Log)：**
    - 在訂單頁面下方顯示操作日誌：
        - `2026-03-02 10:30` - 財務專員王小明 將訂單狀態變更為【已結算】，毛利 24,000。
        - `2026-03-02 11:00` - 庫管專員李小花 將訂單狀態變更為【已扣帳】。扣除項目：NK針劑x2 (庫存: 50 -> 48)。
2. **庫存變動明細表：**
    - 在庫存管理頁面，可以查看每一筆庫存的進出記錄，來源指向具體的訂單編號。
    - *查詢 NK 針劑* -> 看到 `[出庫] 訂單 D2D-005 扣帳，減少 2 支`。
3. **連動儀表板 (Dashboard)：**
    - **財務儀表板：** 當訂單變為「已結算」時，「本日/本月毛利」圖表應立即增加該筆數值。
    - **庫存儀表板：** 當訂單變為「已扣帳」時，「當前庫存價值」應下降，「熱銷產品消耗量」圖表應更新。
