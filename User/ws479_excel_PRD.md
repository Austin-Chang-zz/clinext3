# 網頁應用程式開發提示

## 專案概述

開發一個基於Excel (account1.xlsx)數據結構的醫美診所管理Web應用程式，包含現金帳管理、進貨管理、庫存管理、儀器管理、設定管理及綜合儀表板功能。

---

## 一、數據表結構與功能需求

### 1. 現金帳管理 (Cash Account)

**對應Sheet:** `現金帳`

### 資料欄位:

| 欄位名稱    | 說明                           | 資料類型        |
| ----------- | ------------------------------ | --------------- |
| 訂單編號    | 唯一訂單識別碼 (如: D2D-001)   | 文字            |
| 日期        | 訂單日期                       | 日期            |
| 客戶        | 客戶名稱                       | 文字 (下拉選單) |
| 療程編號    | 療程唯一識別碼                 | 文字            |
| 療程種類    | 醫療/美容/醫美                 | 文字 (下拉選單) |
| 療程項目    | 打針/做臉                      | 文字 (下拉選單) |
| 種類 (針劑) | NK/其他                        | 文字 (下拉選單) |
| 藥材        | 使用的藥材名稱                 | 文字 (下拉選單) |
| 訂價        | 原始價格                       | 數字            |
| 折扣        | 折扣百分比                     | 數字            |
| 折扣金額    | 自動計算: 訂價 × 折扣          | 數字            |
| 收入        | 自動計算: 訂價 - 折扣金額      | 數字            |
| 訂單收入    | 該訂單總收入                   | 數字 (彙總)     |
| 成本        | 單項成本                       | 數字            |
| 成本小計    | 該訂單總成本                   | 數字 (彙總)     |
| 毛利        | 收入 - 成本小計                | 數字            |
| 毛利率      | (毛利/成本小計)                | 百分比          |
| 費用名稱    | 護士/計程車等                  | 文字 (下拉選單) |
| 費用金額    | 費用金額                       | 數字            |
| 費用小計    | 該訂單總費用                   | 數字 (彙總)     |
| 訂單毛利    | 訂單收入 - 成本小計 - 費用小計 | 數字            |

### 功能需求:

- 訂單新增/編輯/刪除/查詢
- 自動計算公式實現
- 按訂單編號分組顯示明細
- 支援多筆療程在同一訂單下
- 支援多筆費用在同一訂單下

## 二、儀表板需求 (Dashboard)

### 1. 財務儀表板

- 總營業收入 (今日/本週/本月)
- 總成本
- 總費用
- 總毛利與平均毛利率
- 收入趨勢圖 (折線圖)
- 毛利占比圓餅圖 (按療程種類)
- 客戶貢獻排行 (長條圖)

### 2. 庫存儀表板

- 目前庫存總價值
- 低庫存商品警示清單
- 庫存週轉率
- 熱銷產品排行
- 進貨趨勢圖

### 3. 營運儀表板

- 訂單數量統計
- 客戶來店頻率
- 熱門療程排行
- 費用分析 (護士/計程車占比)

### 4. 即時數據更新

- 當訂單狀態變更時 (已完成 → 已結算 → 已扣帳)，儀表板數據即時更新
- 庫存異動時，庫存儀表板即時反映

## 三、訂單狀態流程設計

### 狀態定義

| 狀態名稱        | 說明                  | 權限      |
| --------------- | --------------------- | --------- |
| 預約待確認      | 客戶已預約，尚未確認  | 前台      |
| 新訂單          | 訂單成立，客戶未到店  | 前台      |
| 進行中          | 客戶到店服務中        | 醫護人員  |
| 待回診/週期中   | 週期療程未完 (如 1/5) | 自動/手動 |
| 已完成 (服務面) | 服務操作完畢          | 醫護人員  |
| 已結算          | 財務審核，計算毛利    | 財務人員  |
| 已扣帳          | 扣除庫存              | 庫管人員  |
| 已取消          | 訂單取消              | 管理人員  |

### 狀態轉換機制

- **按鈕驅動**：每個狀態變更使用獨立按鈕
- **確認彈窗**：顯示即將變更的內容與影響
- **權限控制**：不同角色可見不同按鈕
- **庫存扣帳彈窗**：顯示即將扣除的品項、數量、庫存前後對比
- **操作日誌**：記錄所有狀態變更歷史

## 四、技術規格建議

### 前端 (Frontend)

- React / Vue.js / Next.js
- 響應式設計 (RWD)
- 圖表庫: Chart.js / ECharts / D3.js

### 後端 (Backend)

- Node.js (Express) / Python (Django/Flask) / PHP (Laravel)
- RESTful API / GraphQL

### 資料庫 (Database)

- PostgreSQL / MySQL / MongoDB
- 需支援交易處理 (Transaction) 確保庫存一致性

### 部署 (Deployment)

- 雲端平台: AWS / Google Cloud / Azure / 本地伺服器
- 支援 Docker 容器化部署

## 五、交付成果

1. **完整原始碼** (前端 + 後端)
2. **資料庫 Schema** (含範例數據)
3. **API 文件** (Swagger/Postman)
4. **使用者手冊** (操作說明)
5. **部署指南** (安裝與設定步驟)
6. **測試案例** (單元測試/整合測試)

## 六、注意事項

- 所有公式需在後端或前端準確實現
- 庫存扣減必須使用資料庫交易 (Transaction) 確保一致性
- 儀表板數據需定期更新或即時更新
- 支援多使用者權限管理 (Role-based Access Control)
- 需有備份與還原機制
